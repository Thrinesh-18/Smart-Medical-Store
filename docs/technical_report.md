# Technical Report: Smart Medical Store IR System

## Abstract
This work presents a production-oriented Information Retrieval system for medicine discovery in local pharmacy contexts. The system combines lexical retrieval (BM25/TF-IDF) and semantic retrieval (Sentence Transformers + FAISS) with query understanding modules such as typo correction, medical alias expansion, and symptom mapping. It supports explainable ranked output and operational analytics, and is implemented as a modular FastAPI + React stack.

## Introduction
Medicine search in real pharmacy systems is challenging because users often submit misspelled queries, brand names instead of generic names, and symptom-oriented natural language statements. Pure keyword search is brittle, while pure semantic search may lose exact pharmaceutical constraints. Therefore, a hybrid IR architecture is proposed.

## Problem Statement
Design and implement an end-to-end retrieval engine that accurately returns relevant medicines for lexical, symptom-based, and semantic queries, while exposing score transparency and practical filtering capabilities.

## Objectives
- Build high-recall lexical retrieval using BM25 and query expansion
- Build contextual semantic retrieval with MiniLM embeddings and vector index
- Fuse lexical and semantic results via weighted hybrid ranking
- Provide explainability for each ranked result
- Deliver production API/UI, evaluation, and deployment assets

## Literature Survey (Condensed)
- **BM25 (Robertson et al.)** remains a robust probabilistic ranking baseline in domain-specific corpora.
- **Dense retrieval** with transformer embeddings improves semantic recall for natural language queries.
- **Hybrid retrieval** improves robustness under vocabulary mismatch and exact matching constraints.
- **Evaluation metrics** such as MAP, MRR, and NDCG remain standard for retrieval quality assessment.

## Methodology
### Data Layer
Synthetic but realistic dataset of 600 records with medicine attributes, symptom tags, aliases, and inventory metadata.

### NLP Pipeline
1. Lowercasing and punctuation normalization  
2. Tokenization and stopword removal  
3. Lemmatization + stemming  
4. RapidFuzz spell correction  
5. Query expansion via aliases, symptom maps, and WordNet synonyms  

### Retrieval
- **Lexical branch**: BM25 over medicine text profile.
- **Semantic branch**: all-MiniLM-L6-v2 embeddings indexed in FAISS (`IndexFlatIP`).
- **Hybrid score**:
  `final = alpha * norm_bm25 + beta * norm_semantic`

### Explainability
Each result emits:
- matched keywords
- semantic score
- alias match flag
- symptom overlap
- retrieval source

## System Design

### Complexity
- BM25 scoring: `O(N * avg_doc_len)` per query
- FAISS flat index: `O(N * d)` for brute-force cosine/IP search
- Hybrid merge/rank: `O(N log N)` sorting
- With caching and ANN alternatives, practical latency remains low for medium corpora.

## Implementation
- Backend: FastAPI, pydantic schemas, rate limiting middleware
- Frontend: React + Tailwind dashboard with search, filters, explanations, analytics
- Data: JSON/CSV/SQLite generated by deterministic script
- Tests: unit + integration + Locust load profile

## Results (Template for Run Output)
